# PRD: SalaryCipher — 链上隐私薪资管理平台

---

## 1. Summary

SalaryCipher 是一个基于 Zama Protocol FHE（全同态加密）技术的企业 HR 薪资管理平台。它让企业能在区块链上完成薪资发放、薪资谈判和合规审计，同时保证薪资数据全程加密，任何第三方（包括区块链节点）均无法读取。平台支持多公司多角色管理，并提供可验证的链上隐私收入证明（RWA）。

## 3. Background

### 背景

区块链上的所有交易默认是公开透明的。任何人只要知道一个钱包地址，就能在链上查到该地址收到的所有转账记录，包括金额和来源。

这对企业薪资场景是一个严重问题：

- 员工之间可以互相查询薪资，造成职场信息不对称和内部矛盾
- 竞争对手可以通过链上数据分析企业的人力成本和团队规模
- 企业无法在保护员工隐私的前提下使用链上支付工具

### 为什么是现在

Zama Protocol 在 2024-2025 年实现了 fhEVM 的生产可用，首次让开发者能在以太坊兼容链上直接对加密数据做计算（而不只是存储）。这意味着：

- 薪资金额可以在加密状态下存储和传输
- 合约可以在不知道具体数字的前提下执行薪资比较、统计等逻辑
- 以上能力在一年前还不可能低成本实现

### 黑客松背景

本产品为 Zama Protocol 黑客松参赛作品，参赛要求：基于 fhEVM 构建可运行的 dApp，完整展示 FHE 技术的隐私特性，需包含智能合约、前端和项目文档。

## 4. Objective

### 目标

为企业提供一个可在区块链上安全发放薪资的工具，让薪资数据对外部完全不可见，同时满足合规审计需求。

### 对用户的价值

- 雇主：用熟悉的 USDC/ETH 支付薪资，无需担心薪资数据泄露
- 员工：收到链上薪资的同时，个人收入对同事和外部完全保密
- HR：可管理员工薪资和审计报告，无需接触原始数据

### 对黑客松的价值

- 充分展示 Zama FHE 的加密存储 + 密文计算两类能力
- 覆盖真实企业场景，具备落地可能性，不是纯技术 demo

### 成功指标（hackathon 阶段）

| 指标               | 目标                            |
| ------------------ | ------------------------------- |
| FHE 加密字段覆盖数 | ≥ 8 个不同数据类型              |
| 可演示的核心流程   | 发薪 / 谈判 / 证明 均可完整跑通 |
| 合约部署           | 成功部署到 Zama 测试网          |
| 前端可用性         | 评审可独立操作完成主流程        |

## 5. Market Segments

### 主要用户

**Web3 原生企业**

- 已在使用加密货币发放薪资或奖金的团队
- 关心链上隐私，不希望薪资结构被外部分析
- 有一定 Web3 操作基础，能使用 MetaMask 等钱包

**传统企业的加密薪资试验者**

- 希望探索链上支付但担心数据透明度问题
- 需要合规审计能力（税务/劳动法）
- 对员工薪资保密有强需求

### 约束条件

- 用户需要具备钱包使用基础（连接 MetaMask、签名交易）
- 平台代币当前为 ConfidentialERC20，提取时会有链上可见性，需用户知情
- 初期部署在 Zama 测试网，主网上线取决于 Zama Protocol 进度

## 6. Value Propositions

### 用户需要完成的事（Jobs to be Done）

1. 定期给员工发薪，不想让同事互相知道薪资
2. 调整已入职员工薪资时，不想暴露公司报价和员工期望
3. 员工需要向银行/房东证明收入，但不想透露具体数字
4. 企业需要内部薪资公平审计，但不想暴露个人数据

### 我们带来的收益

- 薪资金额、接收钱包、资金池余额全程链上加密
- 合约在密文下执行统计，HR 看到的是结论而不是原始数据
- 员工可生成「月薪 ≥ X」的链上隐私证明，可铸造为 NFT 提交给第三方

### 我们帮助避免的痛点

- 避免员工通过 Etherscan 查到同事薪资
- 避免竞争对手分析企业人力成本结构
- 避免调薪谈判中一方被迫透露底线

### 相比竞品的优势

| 方案                         | 问题                     |
| ---------------------------- | ------------------------ |
| 普通 ERC20 转账              | 金额完全公开             |
| Tornado Cash 类混币器        | 合规风险，不适合企业场景 |
| 中心化 HR 系统（Workday 等） | 数据由平台掌控，非自托管 |
| ZK 方案（如 Aztec）          | 需要迁移到新链，生态割裂 |

SalaryCipher 在现有公链上直接实现隐私，无需桥接，对用户透明。

## 7. Solution

### 7.1 用户流程

**首次登录流程：**

```
连接钱包
  ↓
判断是否有公司（创建的 + 加入的）
  ├── 无公司 → 引导页（创建公司 / 等待邀请）
  ├── 1家公司 → 直接进入主界面
  └── 多家公司 → 弹出公司选择框（含角色标记）→ 选择后进入
```

**发薪流程（Owner）：**

```
财务页充值 USDC → 转为加密余额
  ↓
人员页添加员工（地址 + 月薪加密存储）
  ↓
系统按周期自动触发发薪（合约执行，全程密文）
  ↓
员工平台余额增加（加密）
  ↓
员工可选择提取为 USDC/ETH（此时链上可见，UI 提示）
```

**薪资谈判流程：**

```
Owner 或员工本人选择已入职且已有月薪的员工，创建调薪谈判
  ↓
发起方提交第一份加密报价：
  - Owner 提交 employerOffer
  - Employee / HR 本人提交 employeeAsk
  ↓
另一方提交对应的加密回应报价
  ↓
合约密文下比较
  ↓
双方收到结论：「匹配」或「不匹配」
  ↓
匹配成功后，Owner 将 employerOffer 应用为员工新的正式月薪
```

**薪资证明流程（Employee）：**

```
选择公司 + 证明类型（如月薪 ≥ 5000 USDC）
  ↓
触发签名 → 合约在密文下执行比较
  ↓
生成链上证明（ebool 结果 + 元数据）
  ↓
可复制证明链接 / 铸造为 NFT / 分享
```

### 7.2 核心功能

#### 功能一：多公司多角色管理

- 用户可创建多个公司，自动成为 Owner
- 角色体系：Owner（全部权限）/ HR（管理员工，不可操作财务）/ Employee（只看自己数据）
- Topbar 公司切换器：切换后菜单和数据按新角色全量刷新
- 无权限菜单项直接隐藏（而非灰显）

#### 功能二：加密薪资发放

- 平台使用 ConfidentialERC20 Token 作为内部结算单位
- 员工月薪、历史记录、累计待发均以 `euint256` 加密存储
- 发薪周期：每月1日 / 每月15日 / 每两周（可配置）
- 员工离职时立即按天计算比例结算当期薪资
- 提取支持 USDC / ETH，提取时 UI 明确提示链上可见风险

#### 功能三：薪资谈判

- 定位为已入职员工的加密调薪谈判，不用于新员工入职前 offer
- 只能对公司内已存在且已有月薪的 HR / Employee 发起谈判，不能对 Owner 发起
- Owner 可为任意 HR / Employee 发起谈判，Employee / HR 只能为自己发起谈判
- Owner 提交 `employerOffer`，表示公司愿意调整到的新月薪
- 员工提交 `employeeAsk`，表示员工希望调整到的新月薪
- 双方谁先提交都可以，另一方随后提交回应报价
- 合约密文下执行 `employeeAsk <= employerOffer`，输出 `ebool`
- 双方界面只显示「Match」或「No Match」，不暴露双方原始报价
- 匹配成功后，Owner 可将 `employerOffer` 应用为员工新的正式月薪
- 匹配失败时原月薪不变，双方都可开启新一轮谈判

#### 功能四：薪资公平审计

- Owner/HR 触发，合约在密文下计算薪资分布
- 输出聚合性结论（区间、比值），不暴露个体数据
- 报告可下载，含链上哈希作为可验证凭证

#### 功能五：RWA 薪资证明

- 证明类型：月薪超过 X / 月薪在 X~Y 之间 / 过去 N 月持续有收入
- 合约在密文下执行比较，结果为 `ebool`
- 证明含：结论摘要 + 链上哈希 + 有效期
- 支持铸造为 NFT，可提交给第三方 DeFi 协议 / 现实机构验证

#### 功能六：资金管理（仅 Owner）

- 充值：USDC/ETH → 平台加密 Token（两步签名：approve + deposit）
- 提取：加密 Token → USDC/ETH（含链上可见警告）
- 公司资金池健康度：余额（加密）+ 可撑月数 + 颜色进度条
- 资金不足时 Dashboard 显示警告 Banner

### 7.3 技术实现

**智能合约（Solidity + fhEVM）：**

| 合约                  | 职责                             |
| --------------------- | -------------------------------- |
| `CompanyRegistry`     | 公司注册、角色管理               |
| `ConfidentialPayroll` | 加密薪资存储、发薪逻辑、离职结算 |
| `SalaryNegotiation`   | 已入职员工加密调薪谈判、密文比较撮合 |
| `FairnessAudit`       | 加密薪资聚合统计                 |
| `SalaryProof`         | 密文比较 + 证明生成 + NFT 铸造   |

**加密数据类型：**

| 字段             | Zama 类型  | 可见权限                     |
| ---------------- | ---------- | ---------------------------- |
| 员工月薪         | `euint256` | 本人 + Owner + HR            |
| 历史薪资金额     | `euint256` | 本人 + Owner + HR            |
| 累计待发薪资     | `euint256` | 本人 + Owner + HR            |
| 奖金金额         | `euint256` | 本人 + Owner + HR            |
| 公司资金池余额   | `euint256` | 仅 Owner + HR                |
| 员工接收钱包地址 | `eaddress` | 本人 + Owner                 |
| 调薪谈判报价（双方） | `euint256` | 合约内部，双方均不可见原始值 |
| 证明比较结果     | `ebool`    | 员工选择性披露               |

**前端技术栈：**

- React + TypeScript
- wagmi / viem（链交互）
- fhEVM JS SDK（加密输入、解密请求）
- TailwindCSS + 自定义 Design Token

**部署目标：** Zama Devnet / Sepolia 测试网

## 8. Release

### V1（黑客松提交版）

**目标：** 可完整演示核心 FHE 功能，评审可独立操作

**包含功能：**

- 连接钱包 + 公司创建/加入/切换
- 员工管理（添加/编辑/移除 + 离职结算）
- 加密薪资发放（手动触发，非自动定时）
- 薪资谈判（已入职员工调薪 + 双方加密报价 + 密文匹配）
- 薪资公平审计（基础版，输出结论）
- RWA 薪资证明（生成 + 展示，NFT 铸造可选）
- 资金充值/提取
- 完整前端 UI

**不包含（推迟到 V2）：**

- 自动定时发薪（合约 Keeper/Scheduler）
- x402 触发支付
- 薪资证明对外 API
- 移动端适配
- 多语言（先英文）

### V2（黑客松后迭代）

- 自动定时发薪（Chainlink Automation 或类似方案）
- x402 HTTP 触发支付
- 薪资证明开放验证 API
- 多链部署
- 移动端适配
